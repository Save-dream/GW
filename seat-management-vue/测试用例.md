# 工位管理系统测试用例

## 1. 测试目的

验证工位管理系统的所有功能模块是否正常工作，包括：
- 系统登录与权限验证
- 场地管理（增删改查）
- 楼层管理（增删改查）
- 区域管理（增删改查）
- 工位管理（增删改查、分配、更换）
- 绑定管理（人员与工位绑定）
- 操作日志（查看、搜索）
- 系统设置（配置项管理）

## 2. 测试环境

- 前端：Vue 3 + TypeScript + Vite
- 后端：Django REST Framework
- 浏览器：Chrome 144+
- 分辨率：1920x1080
- 网络：本地开发环境

### 7.1 测试环境配置

- 后端：http://localhost:8000/api/

### 7.2 测试数据

- 测试用户：admin/123456
- 测试场地：至少2个场地
- 测试楼层：每个场地至少2个楼层
- 测试区域：每个楼层至少2个区域
- 测试工位：每个区域至少5个工位
- 测试人员：至少5个人员

### 7.3 常见问题排查

1. **API调用失败**：检查后端服务是否运行，检查请求参数格式是否正确
2. **页面无响应**：检查浏览器控制台是否有错误，检查Vue组件是否正确渲染
3. **数据不同步**：检查前端是否正确调用API，检查后端是否正确处理请求
4. **权限问题**：检查登录状态，检查token是否有效
